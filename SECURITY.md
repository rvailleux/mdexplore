# Security Policy

## Supported Versions

We currently provide security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability within mdexplore, please send an email to [security@example.com]. Please **DO NOT** create a public GitHub issue for security vulnerabilities.

We take security seriously and will respond to your report within 48 hours. We appreciate your responsible disclosure and will work with you to understand the scope of the issue and ensure a proper fix.

## What to Expect

1. **Acknowledgment**: We will acknowledge receipt of your vulnerability report within 48 hours
2. **Assessment**: We will assess the impact and severity of the issue
3. **Timeline**: We will provide an estimated timeline for a fix
4. **Fix**: We will develop and test a fix
5. **Disclosure**: We will coordinate with you on public disclosure timing

## Security Best Practices for Users

- Keep mdexplore updated to the latest version
- Only open markdown files from trusted sources
- Be cautious when using mdexplore with files that contain sensitive information
- Report any suspicious behavior immediately
